# BrokerMenuBar (E8) — build, run & verify

A SwiftUI `MenuBarExtra` menu-bar agent that is a pure projection of the Bazel
Broker daemon: glance at running builds and kill them from the menu bar. All logic
lives in the daemon; this app renders state and POSTs intents.

- macOS 14+ (the modern `@Observable` Observation stack + `MenuBarExtra(.window)`).
- Lives **outside** the Go module (own dir, own Xcode project). The Go module never
  compiles these Swift files.
- The Xcode project is **generated** from `project.yml` by [XcodeGen]; `project.yml`
  is the source of truth. `BrokerMenuBar.xcodeproj/` is git-ignored.

## Build / test / launch (via xcodebuildmcp-cli)

```bash
cd apps/MenuBar
xcodegen generate                                  # regenerate the .xcodeproj from project.yml

# Build (ad-hoc signed, no team/notarization):
xcodebuildmcp macos build  --project-path "$PWD/BrokerMenuBar.xcodeproj" --scheme BrokerMenuBar

# Test — THE CONTRACT GATE (decodes every testdata/api/*.json + route + render tests):
xcodebuildmcp macos test   --project-path "$PWD/BrokerMenuBar.xcodeproj" --scheme BrokerMenuBar

# Launch the menu-bar agent:
APP=$(find ~/Library/Developer/XcodeBuildMCP -name BrokerMenuBar.app -type d | head -1)
xcodebuildmcp macos launch --app-path "$APP"
```

## Demo against a stub broker (no live E2/E3/E4 needed)

```bash
python3 scripts/stub-broker.py 8765 &     # writes ~/.config/bazel-broker/config.json + serves fixtures
"$APP/Contents/MacOS/BrokerMenuBar"        # eager-connects at launch; menu shows 2 fixture builds
```

The app reads the daemon-owned `config.json` (`$BAZEL_BROKER_CONFIG` →
`$XDG_CONFIG_HOME/bazel-broker/config.json` → `~/.config/bazel-broker/config.json`,
matching E2) for `host`/`port`/`token`, then talks loopback-only with
`Authorization: Bearer <token>`.

## Verification notes

- `xcodebuildmcp` (this CLI version) has `macos build/launch/test/stop` but **no
  `macos screenshot` verb**, and its `ui-automation` tools are iOS-Simulator-only —
  they cannot drive a macOS `NSStatusItem` dropdown (and System-Events AppleScript /
  `screencapture` need TCC grants not present in a headless run). So the menu's
  visual proof is the **`MenuRenderSnapshotTests` ImageRenderer PNG**
  (`docs/menu-connected.png`), and the kill/route proof is `BrokerClientTests`
  (stub URLProtocol) + the live stub-broker connect (2 established sockets).
- The hard correctness gate is `ContractDecodeTests`: it decodes the canonical
  `testdata/api/*.json` through the real Codable models. If those pass, app and
  daemon agree on the wire contract.

## Frozen-contract conformance (internal/api)

- `Build`: `invocation_id`, `worktree`, `worktree_name?`, `targets[]`, `pid`,
  `state` (`running` not `building`; `queued/finished/failed/killed/gone/unknown`),
  `start_time`, `end_time?`, `exit_code`, `source`, `elapsed_ms`, `cache_hit_ratio?`,
  `profile_url?`.
- `GET /builds` → `{"builds":[…]}`. WS `/events` envelope = **exactly** `snapshot`
  (full list) and `build` (upsert by `invocation_id`); ping frames are protocol-level.
- Kill route: **`POST /builds/{invocation_id}/kill`** (bearer). `profile_url` is a
  fully-formed URL the app just `open`s (E4 populates it).
